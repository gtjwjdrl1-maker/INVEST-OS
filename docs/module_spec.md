# 모듈 상세 스펙

> 이 파일이 모든 모듈 구현의 단일 출처(Single Source of Truth)다.
> 스펙과 다르게 구현해야 할 이유가 있으면 먼저 사람에게 확인한다.

---

## M-2-1 · DART 재무제표 기준 적용 (`m2_1_dart_cutoff`)

| 항목 | 내용 |
|---|---|
| Step | 2 — 리스크 관리 |
| 트리거 | 사용자가 종목 선택 후 "분석" 버튼 클릭 |
| 입력 | 종목코드 (6자리), DART API 키 |
| 출력 | 부채비율, 배당성향, CB 잔액, 우발부채 요약 |
| 편입 기준 | 부채비율 ≤ 100%, 배당성향 ≤ 70%, CB 잔액 = 0 |
| 에러 처리 | API 키 없으면 "DART API 키 필요" 안내 + 데모 데이터 표시 |

## M-2-2 · 스트레스 테스트 / 백테스팅 (`m2_2_stress_test`)

| 항목 | 내용 |
|---|---|
| Step | 2 — 리스크 관리 |
| 트리거 | 자산배분 변경 또는 "백테스팅 실행" 버튼 클릭 |
| 입력 | 현재 포트폴리오 배분 비율, 기간 선택 (1y / 3y / 5y) |
| 출력 | 연환산 수익률, MDD, 샤프지수, 리스크 레벨 (5단계) |
| 데이터 소스 | yfinance (해외), pykrx (국내) |
| 에러 처리 | 데이터 없으면 마지막 캐시 결과 표시 |

## M-2-3 · 대체자산 편입 신호 (`m2_3_alt_signal`)

| 항목 | 내용 |
|---|---|
| Step | 2 — 리스크 관리 |
| 트리거 | 매일 오전 9시 자동 체크 (또는 수동 새로고침) |
| 편입 기준 — 리츠ETF | 배당수익률 스프레드 ≥ 1.5%p (10년물 국채 대비) |
| 편입 기준 — 금ETF | 실질금리 ≤ 0% AND DXY 약세 (3개월 -3% 이하) |
| 편입 기준 — BTC | 현재가 ≥ 200일 이동평균 |
| 출력 | 각 자산별 ✓ 편입 유지 / ✗ 편입 불가 / ⚠ 재검토 배지 |
| 데이터 소스 | yfinance (BTC-USD, GLD, VNQ), FRED API (실질금리, DXY) |

## M-3-1 · AI 찬반 토론 (`m3_1_ai_debate`)

| 항목 | 내용 |
|---|---|
| Step | 3 — AI 투자심의위원회 |
| 트리거 | "심의 시작" 버튼 클릭 |
| 입력 | 검토 종목, 검토 비중 (%) |
| 출력 | 찬성 위원 의견 (밸류에이션), 반대 위원 의견 (리스크), Half-Kelly 최적 비중 |
| AI 모델 | Anthropic Claude (claude-sonnet-4-6) |
| Half-Kelly 계산 | Full Kelly = (승률 × 배당 - 패율) / 배당, Half-Kelly = Full Kelly / 2 |
| 에러 처리 | API 키 없으면 데모 의견 텍스트 표시 |

## M-3-2 · KIS 주문 연동 (`m3_2_kis_order`)

| 항목 | 내용 |
|---|---|
| Step | 3 — AI 투자심의위원회 |
| 트리거 | "주문 실행" 버튼 클릭 (수동 확인 후) |
| 입력 | 종목코드, 수량, 주문유형 (시장가/지정가), 매수/매도 |
| 출력 | 주문 접수 번호, 체결 상태 |
| API | 한국투자증권 Open API (KIS Developers) |
| 안전장치 | 주문 전 확인 다이얼로그 필수 표시, 1회 주문 한도 설정 |
| 에러 처리 | API 키 없으면 "모의투자 모드" 로 전환 |

## M-4-1 · 비중 모니터링 (`m4_1_weight_monitor`)

| 항목 | 내용 |
|---|---|
| Step | 4 — 집행·모니터링 |
| 트리거 | 실시간 (30초마다 자동 새로고침) 또는 수동 |
| 출력 | 현재 배분 vs 목표 배분 차이, 리밸런싱 필요 종목·금액 |
| 임계값 | 목표 대비 ±5%p 이상 이탈 시 알림 |
| 알림 채널 | 앱 내 배너 + 카카오톡 (API 연결 시) |

## M-4-2 · 자동 브리핑 (`m4_2_briefing`)

| 항목 | 내용 |
|---|---|
| Step | 4 — 집행·모니터링 |
| 스케줄 | 오전 9:00 (장 시작 전), 오후 4:00 (장 마감 후) |

| 에러 처리 | API 실패 시 앱 내 브리핑 패널에만 표시 |
